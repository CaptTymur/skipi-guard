# AGENTS.md — инфра-гейт (максимальная строгость)

Ты работаешь в механизме защиты. Ошибка здесь ослабляет гейт для всех репо Skipi — радиус поражения максимальный.

## Правила гейта

- **Изменения — только через PR; мержит человек.** Прямых пушей в main нет.
- **Не ослаблять проверки без явного решения человека.** Расширение allowlist (`configs/homes/*` → `allowed_file_patterns`), scope-логики или override-токенов, сужение `protected_paths` — только отдельным PR с обоснованием каждого пути. Прецеденты: ветка `guard/broker-allow-map` (расщепление по решению 15.07), инцидент SKI-INC-2026-07-15 override-token-bypass.
- **Канон хука — в репо.** Задеплоенные pre-push хуки домов обязаны совпадать с закоммиченным здесь источником (шаблон/генератор); enforcement-логика, живущая только в `.git/hooks/`, запрещена — она вне диффов и уже тихо дрейфовала (прецедент shipmgmt, supervision-находка 15.07).
- `--no-verify` запрещён; force-push (включая `--force-with-lease`) запрещён.

## Общий протокол (одинаков во всех репо Skipi)

- Одна задача = один worktree = одна сессия. Задача закрыта — сессия закрывается; новая задача = свежая сессия.
- Минимальный дифф: меняй только то, что требует задача.
- Bugfix = сначала failing-test, воспроизводящий баг → фикс → зелёный прогон. Фикс без failing-test'а не принимается.
- После работы: commit + push.
- `--no-verify` запрещён.
- systemd, деплой, прод-сервер и флот ботов — вне скоупа; нужно — handoff в штаб (`skipi-ops/handoffs/`).
- Инцидент (регрессия, потеря данных) → SKI-INC-NNN в `skipi-ops/incidents/`.
- Сессия запускается из папки этого репо (`cd <репо> && claude`), не из корня. Если этого файла нет в списке `/memory` — сессия поднята неправильно: остановись и сообщи человеку.
- Обмен с другими агентами — через артефакты на диске (коммиты, handoffs, verify-отчёты), не копипастом между окнами.
